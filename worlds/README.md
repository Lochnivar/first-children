## World Organization

Worlds are organized under `hypatia/worlds/` to keep them together with framework-level documentation. Each world has:
- Full world bible with all details (characters, world-building, plots, etc.)
- Framework summary entry in `framework/worlds/[world-name].md`

## Adding New Worlds

When adding a new world:

1. Create a new directory under `hypatia/worlds/`
2. Create a framework summary entry `hypatia/worlds/[world-name].md`
3. Create a `README.md` in the world directory stating it's subordinate to the framework
4. Create a `CONSISTENCY_CHECK.md` documenting how it conforms
5. Reference the [framework](../FRAMEWORK.md) in your documentation
6. Follow the structure established by existing worlds

## World Structure Template

Each world should contain:
- `README.md` - World overview and navigation
- `CONSISTENCY_CHECK.md` - Framework conformance documentation
- World-specific content (characters, world-building, plots, etc.)
